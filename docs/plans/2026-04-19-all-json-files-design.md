# `--all` 输出文件节点设计

## 背景

当前 `--all` 已经支持输出指定位置下的目录树 JSON，但节点里只有目录，没有文件。用户希望 `--all` 不仅展示文件夹，还要展示每个目录下的直接文件及其大小。

## 目标

- 在 `--all` 输出的每个目录节点中新增 `files` 数组。
- `files` 中每项至少包含 `path` 和 `bytes`。
- 目录节点继续保留 `children` 数组表示直接子目录。
- 目录节点 `bytes` 继续表示该目录自身文件与所有后代目录文件的递归汇总。
- `files` 与 `children` 都按大小从大到小排序。
- `--min-size` 同时作用于 `files` 和 `children`。
- `--limit` 在 `--all` 模式下继续表示“每层数量”，分别限制每个目录的直接文件数量和直接子目录数量。

## 方案

采用“扫描阶段保留文件节点，输出阶段按父目录挂接”的方案：

1. 扩展扫描上下文，除了目录集合外，再保留一个文件集合。
2. 解析 NTFS MFT 或平台 API 文件时，把文件名、父目录、逻辑大小、分配大小保存为文件节点。
3. `--all` 输出阶段同时构建：
   - 目录到子目录的索引
   - 目录到直接文件的索引
4. 目录 `bytes` 继续沿用当前递归汇总逻辑；文件 `bytes` 只取单文件大小。
5. 输出 JSON 时，每个目录节点写出：
   - `path`
   - `bytes`
   - `files`
   - `children`

## 边界

- NTFS 硬链接仍按文件 FRN 去重；若一个文件存在多个名字或多个父目录，当前只保留扫描记录中选中的那一个名字/父目录。
- 根节点始终输出；`--min-size` 不会过滤根节点本身。
- `files` 只表示当前目录的直接文件，不递归展开到下层目录。

## 受影响文件

- `src/model.h`：新增文件节点和文件集合。
- `src/aggregate.c`：在扫描聚合阶段保留文件节点。
- `src/platform_scan.c`：平台 API 路径为文件节点补齐文件名。
- `src/output_json.c`：为 `--all` 构建目录/文件双索引并输出 `files`。
- `README.md`：更新 `--all` 的 JSON 结构说明和示例。

## 验证

- 构建 Release x64。
- 运行 `--all` 并确认根节点含 `files` 字段。
- 验证 `files` 和 `children` 都按大小降序。
- 验证 `--limit` 会分别限制每层的 `files` 数量和 `children` 数量。
- 验证 `--min-size` 会同时过滤文件和子目录。
